Career Narrative

Where I started

My first technical writing job was at Wynne Systems, writing ERP documentation and managing releases in MadCap Flare. I learned that documentation is only as good as the process behind it, and most processes were broken. I started fixing them.

Learning to own the whole lifecycle

At Bird, I wrote the user guide for the company's first retail e-scooter. Cross-disciplinary project. Brand, Retail, Vehicle, all in the same room. I migrated 500+ content pieces from Dozuki to Zendesk ahead of deadline. Learned that good docs require understanding the product at a level most writers never reach.

Scaling content architecture

At Attentive, I created help center articles and integration guides for an SMS marketing platform. Built single sourcing and modular writing into the content architecture. UX Writing certified, because the boundary between documentation and product design is thinner than most people think.

Rebuilding everything (Redwood Software)

Redwood was where I went from writer to engineer. Inherited a Finance Automation help site with weekly CS escalations. Audited the source code with Claude Code. Restructured ~200 articles. Built interactive product simulations, a PDF pipeline, role-aware help, and a new navigation architecture, all inside MadCap Flare, an enterprise tool that constrains everything. The escalations stopped. A customer success agent told me the mini-TOC was "much better than what we originally suggested."

Building the future (CoSH + AI Content Engineering)

December 2025. Started building Chronicles of the Six Houses, an AI-driven narrative RPG with 122 handcrafted NPCs, six houses, and a custom LLM quest-generation pipeline. Structured JSON output validation. 43 narrative state flags. Programmatic hallucination prevention. Five-layer generation pipeline. I'm not just documenting systems anymore. I'm building the systems that generate the content.
